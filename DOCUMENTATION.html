Core

main.jsx

Le point de départ de toute l'app. C'est le premier fichier qui s'exécute.

C'est quoi ce fichier ?

Il démarre l'app. Il prend le fichier index.html (qui a juste une div vide #root) et y injecte toute l'application React. Il configure aussi deux trucs globaux : le cache et le routeur.

Monter l'app dans la page

React a besoin d'un point d'ancrage dans le HTML. createRoot prend la div #root et React prend le contrôle de tout ce qu'il y a dedans.

createRoot(document.getElementById('root')).render(<App />)

QueryClient — le gestionnaire de cache global

TanStack Query gère tous les appels API de l'app. On crée un QueryClient une seule fois ici. Les options définies ici s'appliquent à tous les hooks par défaut.

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      gcTime: 1000 * 60 * 60 * 24, // garde le cache 24h
      retry: false,               // pas de retry auto si l'API plante
      refetchOnWindowFocus: false, // pas de rechargement quand on revient sur l'onglet
    }
  }
})

PersistQueryClientProvider — cache persistant

Ce composant wrape toute l'app et sauvegarde le cache TanStack dans le localStorage. Résultat : si tu fermes et rouvres l'app, les données sont déjà là sans refaire un appel API.

const persister = createSyncStoragePersister({
  storage: window.localStorage // sauvegarde dans le navigateur
})

BrowserRouter — le routeur

Permet à React de gérer plusieurs pages (Accueil, Matchs, Résultats, Classement) sans recharger le navigateur. L'URL change mais c'est React qui gère l'affichage.

App.jsx

La structure de base de l'app — navbar fixe + pages qui changent selon l'URL.

C'est quoi ce fichier ?

App.jsx est le composant racine. Il affiche la navbar en permanence en haut, et en dessous il affiche la bonne page selon l'URL. C'est ici que les routes sont définies.

Les routes

Chaque <Route> associe une URL à un composant. Quand tu vas sur /matchs, React affiche MatchAVenir. Pas de rechargement — tout se passe côté client.

<Routes>
  <Route path="/"           element={<Accueil />}    />
  <Route path="/matchs"     element={<MatchAVenir />} />
  <Route path="/resultats"  element={<Resultat />}   />
  <Route path="/classement" element={<Classement />} />
</Routes>

Animation de transition entre pages

Le key={location.pathname} sur la div dit à React de recréer le composant à chaque changement de page. Ça déclenche l'animation CSS page-transition définie dans App.css.

<div key={location.pathname} className="page-transition">
  // recréé à chaque changement d'URL → animation de fondu
</div>

matchClass() — helper pour les classes CSS

Petite fonction qui construit le nom de classe CSS d'un élément selon si l'équipe a gagné ou perdu. Ça évite des conditions imbriquées répétées partout dans le JSX.

function matchClass(base, isWinner, isLoser) {

  if (isLoser)  return `${base} ${base}--loser`

}

Navigation entre les jours

dayOffset représente le décalage en jours par rapport à aujourd'hui. 0 = aujourd'hui, 1 = demain. On ne peut pas aller avant aujourd'hui (minimum 0). La date est recalculée à chaque changement.

const [dayOffset, setDayOffset] = useState(0)

function getTargetDate(offset) {
  const d = new Date()
  d.setDate(d.getDate() + offset)
  return d.toISOString().split('T')[0] // ex: "2024-01-15"
}

Préchargement des jours adjacents

Une fois les matchs du jour chargés, on précharge en arrière-plan les matchs de demain (et d'hier si applicable). Quand l'utilisateur navigue, les données sont déjà là.

useEffect(() => {
  if (matchesLoading) return
  const run = async () => {
    await new Promise(r => setTimeout(r, 2000)) // 2s de délai pour ne pas surcharger l'API
    await prefetchMatchesForDate(queryClient, getTargetDate(dayOffset + 1))
  }
  run()
}, [dayOffset, matchesLoading])

NewsCarousel — carousel automatique

Affiche les actualités 3 par 3. Un setInterval change de page toutes les 10 secondes automatiquement. Il se nettoie via le return du useEffect pour éviter les fuites mémoire.

useEffect(() => {
  if (loading || pages <= 1) return
  const id = setInterval(() => setPage(p => (p + 1) % pages), 10000)

Hook / Utilitaire

localCache.js

Système de cache manuel dans le localStorage avec durée d'expiration (TTL).

Pourquoi ce fichier existe ?

TanStack Query cache les données en mémoire, mais ce cache disparaît au rechargement. Ce fichier ajoute un cache dans le localStorage — les données survivent à la fermeture et au rechargement de la page. Chaque entrée a une date d'expiration.

writeCache() — sauvegarder

Sauvegarde des données avec une durée de vie (ttlMs en millisecondes). L'expiration = maintenant + durée. Le try/catch protège si le localStorage est plein.

export function writeCache(key, data, ttlMs) {
  try {
    localStorage.setItem(PREFIX + key, JSON.stringify({
      data,
      exp:     Date.now() + ttlMs, // quand ça expire
      savedAt: Date.now()         // quand c'a été sauvegardé
    }))
  } catch {} // silencieux si localStorage est plein
}

readCache() — lire (avec vérif expiration)

Lit les données. Si elles ont expiré, elles sont supprimées et on retourne null. Sinon on retourne les données.

export function readCache(key) {
  const entry = _parse(key)
  if (!entry) return null
  if (Date.now() > entry.exp) {
    localStorage.removeItem(PREFIX + key) // nettoyage automatique
    return null
  }
  return entry.data
}

readCacheStale() — lire même si expiré

Retourne les données peu importe leur âge. Utilisé en fallback quand l'API ne répond pas — mieux vaut des données périmées que rien.

export function readCacheStale(key) {
  return _parse(key)?.data ?? null // renvoie même si expiré
}

Hook

useTodayMatches.js

Récupère les matchs d'une date depuis l'API avec cache et mise à jour automatique.

C'est quoi ce fichier ?

Le hook principal de l'accueil. Il fait deux requêtes (ligues européennes + Coupe du Monde), fusionne, déduplique, trie, et gère le cache avec un TTL dynamique. C'est le hook le plus complexe de l'app.

Deux requêtes en une

L'API gratuite ne renvoie pas la WC dans le endpoint global. On fait donc deux appels séparés avec 700ms de délai entre les deux pour éviter le rate limit de l'API.

const euroMatches = await safeFetch(`/api/v4/matches?dateFrom=...&competitions=CL,PL,FL1...`)
await delay(700) // pause pour ne pas se faire bannir
const wcMatches   = await safeFetch(`/api/v4/competitions/WC/matches?dateFrom=...`)

safeFetch() — requête protégée

Une erreur 429 = trop de requêtes. On lève une erreur exprès pour que TanStack garde les dernières données valides plutôt que d'afficher rien. Pour les autres erreurs, on retourne un tableau vide.

if (res.status === 429) throw new Error('429') // TanStack garde les vieilles données
if (!res.ok) return []                              // autre erreur → vide

getRefetchInterval() — fréquence de mise à jour

Si un match est en cours, l'app se rafraîchit toutes les 2 minutes. Sinon toutes les 10 minutes. TanStack appelle cette fonction automatiquement.

function getRefetchInterval(query) {
  const matches = query.state.data ?? []
  const hasLive = matches.some(m => m.status === 'IN_PLAY' || m.status === 'PAUSED')
  return hasLive ? 2 * 60 * 1000 : 10 * 60 * 1000
}

TTL dynamique selon la situation

La durée de vie du cache s'adapte : match en cours = 2min, aujourd'hui sans live = 1h, autre jour = 6h. Pas besoin de rafraîchir souvent les matchs passés.

let ttl
if (!isToday) {
  ttl = 6 * 60 * 60 * 1000  // autre jour → 6h
} else if (hasLive) {
  ttl = 2 * 60 * 1000        // match en cours → 2min
} else {
  ttl = 60 * 60 * 1000       // aujourd'hui sans live → 1h
}

Hook

useMatchs.js

Récupère les matchs d'une compétition et les groupe par journée.

C'est quoi ce fichier ?

Hook générique utilisé par les pages Matchs à venir et Résultats. Il accepte une compétition, un statut (SCHEDULED ou FINISHED) et un ordre (asc ou desc). Il retourne les matchs à plat ET groupés par journée.

groupByMatchday() — regrouper par journée

Prend un tableau de matchs et les range dans un objet par numéro de journée, puis trie les journées et les matchs à l'intérieur de chaque journée.

function groupByMatchday(matches, order = 'asc') {
  const groups = {}
  matches.forEach(match => {
    const day = match.matchday
    if (!groups[day]) groups[day] = []
    groups[day].push(match)
  })
  // résultat : { 1: [match1, match2], 2: [match3] ... }
}

Ce que le hook retourne

4 valeurs : matches (tous les matchs), loading, error, et grouped (matchs regroupés par journée sous forme de tableau [[numéro, matchs], ...]).

return {
  matches: data ?? [],
  loading: isLoading,
  error:   error?.message ?? null,
  grouped: groupByMatchday(data ?? [], order),
}

useStandings.js

Récupère le classement d'une compétition — championnat simple ou multi-groupes.

C'est quoi ce fichier ?

Hook qui appelle l'API pour le classement. Il gère deux cas : les championnats classiques (un tableau unique) et les compétitions avec plusieurs groupes (WC, LDC phase de poules).

Un groupe vs plusieurs groupes

L'API renvoie un tableau standings. Si ce tableau a plus d'un élément, c'est une compétition à groupes (WC Groupe A, B…). Sinon c'est un championnat classique.

if (allGroups.length > 1) {
  // WC ou LDC poules → plusieurs groupes

    table:  allGroups.flatMap(g => g.table ?? []), // tout à plat pour une vue globale
    groups: allGroups.map(g => ({ name: g.group, table: g.table })),

useTeamForm.js

Calcule les 5 derniers résultats (forme) de chaque équipe d'une compétition.

C'est quoi ce fichier ?

Ce hook récupère tous les matchs terminés d'une compétition, puis construit un dictionnaire { teamId → ['W', 'L', 'D', 'W', 'W'] } pour chaque équipe. Utilisé dans la modale et dans le classement.

getResult() — calculer W/L/D

Compare deux scores et retourne le résultat du point de vue de l'équipe dont les buts sont passés en premier.

function getResult(myGoals, theirGoals) {
  if (myGoals > theirGoals) return 'W' // victoire
  if (myGoals < theirGoals) return 'L' // défaite
  return 'D'                          // nul
}

Construction du formMap

Pour chaque match, on calcule le résultat des deux équipes et on ajoute la lettre dans leur tableau. À la fin, .slice(-5) ne garde que les 5 derniers.

matches.forEach(match => {
  const homeResult = getResult(homeGoals, awayGoals) // 'W', 'L' ou 'D'
  const awayResult = getResult(awayGoals, homeGoals)

  formMap[homeId].push(homeResult)
  formMap[awayId].push(awayResult)
})

formMap[id] = formMap[id].slice(-5) // garder seulement les 5 derniers

useWcKnockout.js

Récupère et organise les matchs de la phase finale de la Coupe du Monde.

C'est quoi ce fichier ?

Hook qui récupère tous les matchs WC, filtre ceux de la phase finale (huitièmes, quarts, demies, finale…) et les organise dans un tableau ordonné par tour. Ce tableau sert à dessiner le bracket SVG.

L'ordre des tours + labels français

KNOCKOUT_ORDER définit l'ordre des tours dans le bracket. KNOCKOUT_LABELS traduit les codes API en français lisible.

export const KNOCKOUT_ORDER = [
  'ROUND_OF_32', 'LAST_16', 'QUARTER_FINALS',
  'SEMI_FINALS', 'THIRD_PLACE', 'FINAL',
]

export const KNOCKOUT_LABELS = {
  LAST_16:        'Huitièmes de finale',
  QUARTER_FINALS: 'Quarts de finale',
  FINAL:          'Finale',
}

Construction des rounds

On boucle sur l'ordre des tours et on filtre les matchs qui correspondent. Si un tour n'a pas encore de matchs (phase pas commencée), il n'est pas inclus dans le résultat.

for (const stage of KNOCKOUT_ORDER) {
  const stageMatches = knockout.filter(m => m.stage === stage)
  if (stageMatches.length > 0) {
    rounds.push({ stage, label: KNOCKOUT_LABELS[stage], matches: stageMatches })
  }
}

useNews.js

Récupère des actualités foot depuis l'API GNews, filtre et déduplique.

C'est quoi ce fichier ?

Hook qui appelle l'API GNews, filtre les articles pour ne garder que ceux qui parlent de foot, déduplique par titre, et retourne les 12 plus récents triés du plus récent au plus ancien.

Déduplication par titre normalisé

L'API peut renvoyer le même article avec de légères variations dans le titre. On normalise (minuscules + sans ponctuation) et on utilise une Map — si la même clé apparaît deux fois, la deuxième écrase la première.

const byTitle = new Map()
for (const article of (json.articles ?? [])) {
  const titleKey = article.title?.toLowerCase().replace(/[^\w\s]/g, '').trim() ?? ''
  byTitle.set(titleKey, article) // même titre → écrase le doublon
}
const uniqueArticles = [...byTitle.values()]

Filtrage par mots-clés

Chaque article est vérifié : son titre ou sa description doit contenir au moins un mot-clé foot. Sinon il est ignoré pour éviter les articles hors-sujet.

const FOOTBALL_KEYWORDS = ['football', 'foot', 'ligue', 'league', 'transfert', ...]

.filter(a => {
  const text = `${a.title} ${a.description}`.toLowerCase()
  return FOOTBALL_KEYWORDS.some(kw => text.includes(kw))
})

teamNames.js

Dictionnaire de traduction des noms d'équipes anglais → version courte française.

C'est quoi ce fichier ?

L'API renvoie les noms complets en anglais ("Paris Saint-Germain FC", "Manchester City FC"…). Ce fichier contient un dictionnaire et une fonction translateTeam() utilisée dans tous les composants qui affichent un nom d'équipe.

translateTeam()

Cherche le nom dans le dictionnaire et retourne la version courte. Si le nom n'est pas dans le dictionnaire, retourne le nom original — l'app ne plante jamais pour une équipe manquante.

const TEAM_NAMES = {
  'Paris Saint-Germain FC': 'PSG',
  'Manchester City FC':     'Man City',
  'Real Madrid CF':         'Real Madrid',
  // ... 180+ équipes
}

export function translateTeam(name) {
  return TEAM_NAMES[name] ?? name // si pas trouvé → nom original
}
