Glossaire technique — StatFootix

Tous les termes à connaître pour présenter le projet

API

API REST

Porte d'entrée pour récupérer des données d'un service externe. Tu envoies une URL, le serveur répond en JSON. Ton projet en utilise 3 : football-data.org, api-football.com, ESPN.

API

JSON

Format texte structuré utilisé par toutes les APIs pour répondre. Ressemble à un objet JS : { "home": "PSG", "score": 2 }. Facile à lire pour un humain et pour le code.

Source principale

football-data.org

Source principale pour les matchs, résultats, classements. Free tier = 10 req/min. Lent à mettre à jour les statuts live (1–3 min de délai).

Fallback

api-football.com

Très précis pour le live (minute exacte, mi-temps...). Problème : free tier = 100 req/jour. Utilisé uniquement en fallback dans 4 fenêtres précises pour ne pas gaspiller le quota.

Primaire

ESPN (non-officielle)

API utilisée par ESPN lui-même, sans clé ni quota documenté. Retourne displayClock ("42:00") et le statut précis en temps réel. Source primaire pour le live.

API

Slug

Identifiant court dans une URL. fra.1 = Ligue 1, eng.1 = Premier League, uefa.champions = Champions League. Utilisé pour dire à ESPN quelle compétition tu veux.

API

Quota

Limite de requêtes autorisées. api-football.com = 100 req/jour sur le free tier. À 97% utilisé en une journée → c'est le bug qu'on a corrigé avec les fenêtres de polling.

Réseau

Requête HTTP / fetch

Appel réseau vers une URL. fetch('/espn?slug=fra.1') → navigateur → Netlify → ESPN → JSON en retour.

Live

Polling

Interroger une API à intervalles réguliers. 20s pour ESPN, 60s pour api-football. Différent d'un WebSocket (connexion permanente) — ici tu "poses la question" périodiquement.

JS

setInterval

Fonction JS qui répète du code à intervalles fixes. setInterval(tick, 20_000) → appelle tick() toutes les 20s. Nettoyé avec clearInterval pour éviter les fuites mémoire.

JS

async / await

Syntaxe pour le code asynchrone. await fetch(url) = "attends la réponse avant de continuer". Sans ça, le code passerait à la suite sans avoir les données.

Infra

Netlify Functions

Mini-serveurs hébergés par Netlify. Les navigateurs bloquent les appels directs à ESPN (CORS). La fonction Netlify fait l'intermédiaire : navigateur → ta fonction → ESPN.

Réseau

CORS

Mécanisme de sécurité des navigateurs : un site ne peut pas appeler directement une API d'un autre domaine. D'où les Netlify Functions — elles contournent ça côté serveur.

Stockage

localStorage

Stockage persistant dans le navigateur. Reste là même après rechargement ou fermeture d'onglet. Sert à mémoriser les timestamps des matchs entre deux polls.

Stockage

Timestamp

Nombre = millisecondes depuis le 1er janvier 1970. Date.now() = timestamp actuel. Soustraction de deux timestamps = durée écoulée. Utilisé pour tous les calculs de minutes.

État match

matchState

Ce qu'on stocke par match : kickoffAt (timestamp KO), pausedAt (MT), half2Start (2ème MT), espnClock, espnStatus, espnCapturedAt.

Pourquoi localStorage ? football-data.org peut brièvement retourner un mauvais statut. Grâce aux timestamps locaux, calcMinute continue d'afficher la bonne minute même si l'API est incohérente — les données locales priment sur le statut distant.
React

Hook React

Fonction spéciale React (commence par use) qui gère de l'état ou des effets. useState, useEffect, useRef sont natifs. useLiveMinute est un hook custom.

React

Re-render

React re-exécute un composant pour mettre à jour l'affichage. Déclenché par un changement de state ou de props. calcMinute est appelé à chaque re-render → minute mise à jour.

React

useRef

Stocke une valeur persistante entre renders sans déclencher de re-render. Permet aux fonctions async d'accéder toujours à la version la plus récente de matches.

TanStack

TanStack Query

Librairie qui gère le cache des données serveur. useQuery déclare une requête, la met en cache, refetch automatiquement et notifie les composants quand les données changent.

TanStack

queryKey

Identifiant du cache. ['liveMatches'] = la clé qui identifie les matchs en direct. Deux composants avec la même queryKey partagent automatiquement le même cache.

TanStack

staleTime

Durée pendant laquelle les données sont considérées fraîches. staleTime: 20_000 → pendant 20s après un fetch, pas de refetch même si le composant est revisité.

TanStack

invalidateQueries

Marque le cache comme périmé → refetch immédiat. Utilisé quand ESPN détecte un KO pour forcer useLiveMatches à refetcher football-data.org tout de suite.

TanStack

setQueryData

Écrit directement dans le cache sans requête réseau. Utilisé dans injectLiveMatch pour ajouter un match au cache ['liveMatches'] instantanément.

Live

Fenêtres de polling

Périodes précises où api-football peut poll. Hors fenêtre = 0 requête. Un match = 4 fenêtres courtes (KO, MT, reprise, FT) → ~20 req max au lieu de 120.

Live

stickyLive

Map en mémoire qui garde les matchs même si football-data.org les fait disparaître momentanément. Durée de vie : 3 min. Sert aussi à injecter un match dès qu'ESPN détecte le KO.

Live

Interpolation

Calculer une valeur entre deux points. ESPN donne "42:00" à T+0. À T+30s : 42 + 30/60 = 42'. À T+60s : 43'. Minute en temps réel sans requête supplémentaire.

ESPN

displayClock

Champ ESPN contenant l'horloge du match. "42:00" = 42e minute. "45:00+2:00" = 45+2 en temps additionnel. Format MM:SS normal, MM:SS+MM:SS en prolongation.

ESPN

Statuts ESPN

STATUS_IN_PROGRESS = en cours · STATUS_HALFTIME = mi-temps · STATUS_FINAL = terminé. Plus rapides à se mettre à jour que les statuts football-data.org.

Live

Fuzzy matching

Comparaison approximative de textes. ESPN dit "Inter Milan", FD.org dit "FC Internazionale Milano" → mot commun "inter" détecté → match trouvé. Prefix 5 chars + overlap de mots ≥4 lettres.

Live

Cap de sécurité

Limite max pour éviter les calculs aberrants. ESPN_INTERP_CAP = 30s : si le dernier poll date de plus de 30s (onglet en veille), on affiche la valeur brute plutôt qu'interpoler.

Texte

Normalize

Nettoyer un texte avant comparaison : minuscules + accents retirés (é→e, ü→u...). Sans ça "Atlético" ≠ "atletico" alors qu'ils désignent la même équipe.

La règle d'or : ESPN est ta "montre" — elle donne l'heure exacte. api-football et football-data.org sont les montres de secours si ESPN est en panne. Tout passe par localStorage pour ne pas perdre l'état entre les requêtes.
