Guide du code StatFootix

Chaque fichier, chaque fonction, chaque constante — expliqués simplement.

Stack utilisée

React — interface utilisateur  ·  TanStack Query — fetch + cache  ·  React Router — navigation  ·  Netlify Functions — proxy API sécurisé  ·  football-data.org — données foot  ·  GNews — actualités

Architecture générale

StatFootix est une SPA (Single Page Application). Ça veut dire que la page ne se recharge jamais quand tu navigues — React Router remplace juste le composant affiché à l'écran.

App.jsx — point d'entrée

C'est le fichier racine. Il contient la Navbar (toujours visible), les routes vers chaque page, et le Footer (toujours visible). La clé location.pathname sur la div déclenche l'animation CSS à chaque changement de page.

function App() {
  const location = useLocation()   // l'URL actuelle

  return (
    <>
      <Navbar />                          // barre du haut, toujours là

      <div key={location.pathname} ...>    // key change = animation CSS
        <Routes>
          <Route path="/"              element={<Accueil />} />
          <Route path="/matchs"        element={<MatchAVenir />} />
          <Route path="/resultats"     element={<Resultat />} />
          <Route path="/classement"    element={<Classement />} />
          <Route path="/mentions-legales" element={<MentionsLegales />} />
        </Routes>
      </div>

      <Footer />                          // pied de page, toujours là
    </>
  )
}

Flux de données

Quand un composant a besoin de données (ex: matchs du jour), voilà le chemin complet :

Composant React
→
Hook (useXxx)
→
fetch /api/v4/...
→
Netlify Function
→
football-data.org

La Netlify Function est un mini-serveur intermédiaire. Son rôle : ajouter la clé API secrète à chaque requête avant de la transmettre à football-data.org. Sans ça, la clé serait visible dans le navigateur de n'importe qui.

TanStack Query gère le cache côté navigateur. Si tu reviens sur une page que t'as déjà visitée il y a 5 minutes, les données sont réutilisées sans faire de nouvelle requête.

⚠ Limite API : 10 requêtes/minute

L'API free tier football-data.org est limitée à 10 req/min. Tous les hooks ont un délai de 700ms entre les requêtes et un système de cache intelligent pour éviter de dépasser cette limite.

Accueil Accueil.jsx

La page principale. Elle affiche 4 blocs : le hero avec le widget live, les matchs du jour (navigation jour par jour), les résultats récents (CdM), et le carousel d'actualités.

getDayLabel(offset) — étiquette du jour

Transforme un nombre en texte lisible. 0 → "Aujourd'hui", 1 → "Demain", 2 → "Sam. 14 juin", etc.

function getDayLabel(offset) {
  if (offset === 0) return "Aujourd'hui"
  if (offset === 1) return 'Demain'
  const d = new Date()
  d.setDate(d.getDate() + offset)
  return d.toLocaleDateString('fr-FR', { weekday:'short', day:'numeric', month:'short' })
}

getTargetDate(offset) — date ISO du jour ciblé

Convertit l'offset en date au format YYYY-MM-DD pour l'envoyer à l'API.

function getTargetDate(offset) {
  const d = new Date()
  d.setDate(d.getDate() + offset)
  return d.toISOString().split('T')[0]  // "2026-06-15"
}

formatHour(dateStr) — heure d'un match

Prend une date ISO et retourne juste l'heure : "21:00".

matchClass(base, isWinner, isLoser) — classe CSS dynamique

Retourne la classe CSS à appliquer selon si l'équipe a gagné, perdu, ou ni l'un ni l'autre. Utilisé dans MatchCard pour colorer le vainqueur en jaune et atténuer le perdant.

function matchClass(base, isWinner, isLoser) {
  if (isWinner) return `${base} ${base}--winner`  // jaune
  if (isLoser)  return `${base} ${base}--loser`   // atténué
  return base                                        // normal
}

groupByDay(matches) — regrouper par jour

Prend une liste de matchs et les regroupe par date. Retourne un tableau trié du plus récent au plus ancien. Utilisé dans ResultPanel.

formatDayLabel(dateStr) — "Aujourd'hui" / "Hier" / date

Affiche "Aujourd'hui" ou "Hier" si c'est ces jours-là, sinon la date complète. Utilisé dans la navigation du ResultPanel.

Gère l'état dayOffset (quel jour afficher) et le préchargement des jours adjacents. Quand dayOffset change (tu cliques sur une flèche), targetDate change → useTodayMatches refetch les matchs du nouveau jour.

LiveWidget Accueil.jsx Live

Le petit encadré rouge "EN DIRECT" qui apparaît en haut à droite quand un match est en cours. Si aucun match live → le composant retourne null (invisible).

Il utilise son propre hook useLiveMatches — indépendant des matchs du jour. Ça veut dire que même si tu regardes les matchs de demain, le widget live fonctionne quand même.

function LiveWidget() {
  const matches = useLiveMatches()   // tous les matchs EN DIRECT maintenant
  const live = matches.filter(m =>
    m.status === 'IN_PLAY' || m.status === 'PAUSED'
  )

  // Ticker : force un re-render toutes les 30s pour recalculer la minute
  const [, setTick] = useState(0)
  useEffect(() => {
    if (live.length === 0) return   // pas de live → pas de timer
    const id = setInterval(() => setTick(t => t + 1), 30_000)
    return () => clearInterval(id)  // nettoyage au démontage
  }, [live.length])

  if (live.length === 0) return null  // rien en live → widget caché

  return (
    <div className="accueil__liveWidget">
      ...
      {live.length > 1 && <span>{live.length}</span>}  // badge nb de matchs si 2+
      {live.slice(0, 5).map(match => <MatchCard key={match.id} match={match} noWinnerLoser />)}
    </div>
  )
}
Pourquoi le ticker ?

calcMinute() est une fonction pure qui lit Date.now(). Elle ne se recalcule que lors d'un re-render. Sans le ticker, la minute resterait figée tant que tu ne changes pas de page. Le setInterval force un re-render toutes les 30 secondes pour mettre à jour l'affichage.

MatchCard Accueil.jsx

La carte d'un match individuel. Réutilisée dans le LiveWidget, les matchs du jour et les résultats récents. Elle s'adapte automatiquement selon le statut du match.

function MatchCard({ match, noWinnerLoser = false }) {
  const isLive     = match.status === 'IN_PLAY' || match.status === 'PAUSED'
  const isFinished = match.status === 'FINISHED'
  const hs  = match.score?.fullTime?.home ?? match.score?.halfTime?.home
  const as_ = match.score?.fullTime?.away ?? match.score?.halfTime?.away

  // Label central selon le statut :
  const label     = isFinished ? 'FT' : !isLive ? formatHour(match.utcDate) : null
  const liveLabel = isLive ? calcMinute(match) : null

  // À venir   → label = "21:00"
  // En direct → liveLabel = "34'" ou "MT"
  // Terminé   → label = "FT"
}

Le prop noWinnerLoser désactive le highlight jaune/atténué (utilisé dans le widget live où tous les matchs sont en cours donc il n'y a pas encore de gagnant).

MatchPanel Accueil.jsx

Le bloc gauche de l'Accueil — affiche les matchs du jour. Si certains matchs sont en cours ou à venir, il cache automatiquement les matchs déjà terminés pour mettre en avant ce qui se passe maintenant.

function MatchPanel({ matches: allMatches, loading }) {
  const active    = allMatches.filter(m => m.status !== 'FINISHED')
  const displayed = active.length > 0 ? active : allMatches
  // S'il y a des matchs actifs → afficher seulement eux
  // Sinon (tout est terminé) → afficher tous les matchs du jour

  // Ticker pour recalculer calcMinute() toutes les 30s si live
  const hasLive = displayed.some(m => m.status === 'IN_PLAY' || m.status === 'PAUSED')
  useEffect(() => {
    if (!hasLive) return
    const id = setInterval(() => setTick(t => t + 1), 30_000)
    return () => clearInterval(id)
  }, [hasLive])
}

ResultPanel Accueil.jsx

Le bloc droit de l'Accueil — affiche les résultats récents de la CdM. Navigation avec les flèches ‹ › pour passer d'un jour à l'autre. dayIndex = 0 est le jour le plus récent.

function ResultPanel({ results, loading }) {
  const grouped = groupByDay(results)      // { "2026-06-15": [matchs], ... }
  const [dayIndex, setDayIndex] = useState(0)  // 0 = jour le plus récent

  const currentDay     = grouped[dayIndex]
  const dayLabel       = currentDay ? formatDayLabel(currentDay[0]) : null
  const currentMatches = currentDay ? currentDay[1] : []

  const canGoBack    = dayIndex < grouped.length - 1  // il y a des jours plus anciens
  const canGoForward = dayIndex > 0                   // il y a des jours plus récents
}

BkCard Match.jsx Bracket

La carte d'un match dans le bracket SVG (phase finale CdM). Affiche les deux équipes avec leur blason, le score si disponible, et la minute si le match est en live.

function BkCard({ m, style, onSelect }) {
  const fin  = m.status === 'FINISHED'
  const live = m.status === 'IN_PLAY' || m.status === 'PAUSED'
  const tbd  = !m.homeTeam?.name   // équipe pas encore connue ("À venir")

  // Score affiché seulement si match terminé ou en cours
  {(fin||live) && <span>{hs??0}</span>}

  // Centre de la carte :
  {live && <span>● {calcMinute(m) ?? 'Live'}</span>}  // "● 34'" ou "● MT"
  {!fin && !live && <span>{_fmtD(m.utcDate)} {_fmtH(m.utcDate)}</span>}
}

useTodayMatches hooks/useTodayMatches.js Hook

Fetch les matchs d'un jour donné. Fait 2 requêtes : une pour les ligues européennes, une pour la CdM (l'API free tier ne les mélange pas dans le même endpoint). Délai de 700ms entre les deux pour éviter le rate limit.

async function fetchTodayMatches(date) {
  // Requête 1 : CL, PL, Ligue 1, LaLiga, Bundesliga, Serie A
  const euroMatches = await safeFetch(
    `/api/v4/matches?dateFrom=${date}&dateTo=${date}&competitions=CL,PL,FL1,PD,BL1,SA`
  )
  await delay(700)  // ← pause anti-rate-limit

  // Requête 2 : Coupe du Monde séparément
  const wcMatches = await safeFetch(
    `/api/v4/competitions/WC/matches?dateFrom=${date}&dateTo=${date}`
  )
  // Déduplique par id + trie par heure de coup d'envoi
}

Cache TTL selon le contexte : autre jour → 6h · Aujourd'hui sans live → 1h · Aujourd'hui avec live → 2min.

safeFetch(url)

Comme fetch normal mais gère les erreurs proprement. Si l'API répond 429 (rate limit), lève une erreur pour que TanStack Query garde les données précédentes au lieu d'afficher une erreur.

prefetchMatchesForDate(queryClient, date)

Précharge les matchs d'un jour en arrière-plan sans bloquer l'UI. Appelé depuis l'Accueil pour précharger le jour suivant et précédent, pour que la navigation entre jours soit instantanée.

getRefetchInterval(query)

Retourne 2min si un match est en cours, 10min sinon. Passé à refetchInterval de TanStack Query.

// Filtre : l'article doit mentionner au moins 1 mot-clé football
.filter(a => FOOTBALL_KEYWORDS.some(kw => text.includes(kw)))

matchUtils.js utils/matchUtils.js

Contient calcMinute(match) — la seule fonction utilitaire partagée entre plusieurs composants. Importée dans Accueil.jsx et Match.jsx.

function calcMinute(match) {
  if (match.status === 'PAUSED') return 'MT'      // mi-temps (status API)
  if (match.status !== 'IN_PLAY') return null

  const elapsed = Math.floor((Date.now() - new Date(match.utcDate)) / 60000)
  // elapsed = minutes écoulées depuis le coup d'envoi prévu

  if (elapsed <= 45) return `${Math.max(1, elapsed)}'`  // 1ère mi-temps normale
  if (elapsed <= 62) return 'MT'                        // pause (API ne renvoie pas PAUSED)

  const half2 = elapsed - 62                            // 2ème mi-temps (~pause 17min)
  if (half2 <= 45) return `${45 + half2}'`             // 46' → 90'
  return `90+${half2 - 45}'`                            // arrêts de jeu 2ème MT
}
Limitation connue

L'API free tier ne renvoie pas match.minute et ne passe pas toujours à PAUSED pendant la mi-temps. Le calcul depuis utcDate est une estimation — précis dans ~85% des cas.

Netlify Functions netlify/functions/

Deux mini-serveurs qui servent de proxy entre l'app et les APIs externes. Leur seul rôle : ajouter la clé API secrète aux requêtes.

football.js — proxy football-data.org

exports.handler = async (event) => {
  // event.path = "/api/v4/competitions/PL/matches"
  const apiPath = event.path.replace(/^\/api/, '')  // "/v4/competitions/PL/matches"
  const url = `https://api.football-data.org${apiPath}${qs}`

  const res = await fetch(url, {
    headers: { 'X-Auth-Token': process.env.API_KEY }  // ← clé secrète côté serveur
  })
  return { statusCode: res.status, body: await res.text() }
}

news.js — proxy GNews

Même principe. La clé GNews est stockée dans process.env.GNEWS_KEY sur Netlify — jamais dans le code ou le navigateur.

netlify.toml — règle de redirection

[[redirects]]
  from   = "/api/*"
  to     = "/.netlify/functions/football/:splat"
  status = 200
// Toute URL commençant par /api/ est interceptée et redirigée vers la fonction
En développement local

Le proxy Netlify n'est pas dispo en local. C'est le proxy Vite (vite.config.js) qui fait le même travail : il redirige /api/* vers api.football-data.org et ajoute la clé depuis le fichier .env.
