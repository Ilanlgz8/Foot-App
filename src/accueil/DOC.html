📄 MatchCard.jsx

Ce fichier contient 3 composants et 2 fonctions utilitaires.
Son rôle principal : afficher une carte de match (logo équipe dom · score/heure · logo équipe ext) dans tous les contextes — matchs du jour, widget live, résultats récents.

Fonction formatHour(dateStr)

Convertit une date UTC en heure affichable en français. Par exemple "2026-06-16T19:00:00Z" devient "21:00" (heure locale).

function formatHour(dateStr) { return new Date(dateStr).toLocaleTimeString('fr-FR', { hour: '2-digit', minute: '2-digit' }) }

Utilisée pour afficher l'heure de coup d'envoi sur les matchs à venir.

Fonction matchClass(base, isWinner, isLoser)

Construit une classe CSS selon si l'équipe a gagné, perdu, ou match nul. Elle prend un nom de classe de base et lui ajoute un modificateur CSS.

matchClass('accueil__matchCardName', true, false) → "accueil__matchCardName accueil__matchCardName--winner" matchClass('accueil__matchCardName', false, true) → "accueil__matchCardName accueil__matchCardName--loser" matchClass('accueil__matchCardName', false, false) → "accueil__matchCardName" (nul ou match en cours)

Appelée 4 fois dans MatchCard : pour le nom et le blason de chaque équipe.

Composant PanelSkeleton

Affiche 3 cartes grises animées pendant que les données chargent — ce qu'on appelle un "skeleton loader". Ça évite un écran vide blanc.

Chaque carte skeleton contient : un cercle gris (blason), une barre grise (nom), un rectangle gris (score), même chose pour l'équipe adverse. La classe CSS .sk applique l'animation de pulsation.

Composant principal MatchCard({ match, noWinnerLoser, tracked, onTrack, espnScore })

La carte d'un match. Affiche trois zones côte à côte :

  • Gauche — blason + nom de l'équipe domicile
  • Centre — la minute live / l'heure / "FT" + le score
  • Droite — blason + nom de l'équipe extérieure

Les 5 props qu'elle reçoit :

  • match — l'objet du match (vient de football-data.org)
  • noWinnerLoser — si true, désactive les styles gagnant/perdant (utilisé dans le widget live)
  • tracked — si ce match est suivi pour les minutes précises
  • onTrack — fonction appelée au clic sur "Suivre". Si null → bouton caché
  • espnScore — le score ESPN { home, away } — mise à jour quasi instantanée

Bloc 1 — Déterminer le statut du match

const isLive = match.status === 'IN_PLAY' || match.status === 'PAUSED' const isFinished = match.status === 'FINISHED'

Deux booléens simples. isLive est true si le match est en cours ou en pause (mi-temps). isFinished est true si terminé. Si ni l'un ni l'autre → match à venir.

Bloc 2 — Le score

const hs = espnScore?.home ?? match.score?.fullTime?.home ?? match.score?.halfTime?.home const as_ = espnScore?.away ?? match.score?.fullTime?.away ?? match.score?.halfTime?.away

On prend ESPN en priorité (quasi temps réel). Si ESPN n'a rien → on prend le score fullTime de football-data.org (~1min de délai). Si pas encore de fullTime → score de mi-temps. Le ?? signifie "si c'est null ou undefined, essaie le suivant".

Bloc 3 — Qui a gagné ?

const homeWins = !noWinnerLoser && isFinished && hs != null && as_ != null && hs > as_ const awayWins = !noWinnerLoser && isFinished && hs != null && as_ != null && as_ > hs

Vrai uniquement si : on n'a pas désactivé les styles ET le match est terminé ET on a un score ET l'équipe a plus de buts. Les deux sont false en cas de match nul.

Bloc 4 — Le texte du centre

const label = isFinished ? 'FT' : !isLive ? formatHour(match.utcDate) : null const liveLabel = isLive ? calcMinute(match) : null

label = "FT" si terminé, l'heure si à venir, null si en cours. liveLabel = la minute calculée (ex: "73'", "MT", "90+2'") si en cours, null sinon.

Bloc 5 — Les classes CSS avec gagnant/perdant

const homeNameCls = matchClass('accueil__matchCardName', homeWins, awayWins) const homeCrestCls = matchClass('accueil__matchCardCrest', false, awayWins)

Pour le nom domicile : winner si homeWins, loser si awayWins. Pour le blason domicile : jamais winner (pas de style gagnant sur les logos), loser si l'équipe adverse a gagné (blason grisé).

Bloc 6 — Le rendu JSX

Trois zones dans la carte : équipe domicile (blason + nom), centre (label + score), équipe extérieure (nom + blason). Puis en dessous le bouton "Suivre" — affiché uniquement si onTrack est fourni et que le match n'est pas terminé.

Composant MatchPanel({ matches, loading, espnScores, trackedIds, onTrack, totalMatchCount })

Affiche la liste des matchs du jour dans le panel gauche de l'accueil. Reçoit tous les matchs et décide lesquels montrer.

Logique d'affichage :

const active = allMatches.filter(m => m.status !== 'FINISHED') const displayed = active.length > 0 ? active : allMatches

Si des matchs sont en cours ou à venir → affiche uniquement ceux-là. Si tous les matchs du jour sont terminés → affiche quand même tous les résultats du jour.

Logique du bouton Suivre : visible si plus de 5 matchs ce jour-là ET que la limite de 5 suivis n'est pas atteinte (ou que le match est déjà suivi).

📄 LiveWidget.jsx

Affiche le widget "EN DIRECT" en haut à droite de l'accueil quand des matchs sont en cours. Totalement indépendant du jour affiché dans le panel principal — il montre toujours les matchs live du moment.

Composant LiveWidget({ espnScores, trackedIds, onRecalibrate })

Les 3 props qu'il reçoit :

  • espnScores — objet { matchId: { home, away } } avec les scores ESPN quasi temps réel
  • trackedIds — Set des IDs de matchs suivis (pour afficher le bon style sur les MatchCards)
  • onRecalibrate — fonction du bouton ⟳ pour recalibrer les minutes si elles dérivent

Bloc 1 — Récupérer les matchs live

const matches = useLiveMatches() const live = matches.filter(m => m.status === 'IN_PLAY' || m.status === 'PAUSED')

useLiveMatches() est appelé dans Accueil.jsx aussi — grâce au cache React Query (même queryKey), aucune requête réseau supplémentaire n'est faite ici. On filtre pour ne garder que les matchs vraiment en cours ou en pause.

Bloc 2 — Si aucun match en direct

if (live.length === 0) return null

Le widget disparaît complètement quand aucun match n'est en direct. React retourne null = rien n'est affiché dans le DOM.

Bloc 3 — L'en-tête du widget

Contient : le point rouge clignotant, le texte "EN DIRECT", le nombre de matchs si plus d'un, et le bouton ⟳ de recalibration (visible uniquement si onRecalibrate est fourni).

Bloc 4 — La liste des matchs

live.slice(0, 5).map(match => <MatchCard ... />)

Maximum 5 matchs affichés dans le widget (cas rare mais possible lors de grosses journées). Chaque match utilise le composant MatchCard avec noWinnerLoser=true (pas de style gagnant/perdant sur des matchs en cours).

📄 ResultPanel.jsx

Affiche les résultats récents dans le panel droit de l'accueil. Permet de naviguer entre les jours avec des flèches ← →.

Fonction groupByDay(matches)

Reçoit un tableau de matchs et les regroupe par date. Retourne un tableau de paires [date, matchs] triées du plus récent au plus ancien.

Entrée : [{ utcDate: "2026-06-16T...", ... }, { utcDate: "2026-06-15T...", ... }] Sortie : [["2026-06-16", [...matchs]], ["2026-06-15", [...matchs]]]

m.utcDate.slice(0, 10) extrait uniquement la partie date "2026-06-16" depuis la chaîne complète "2026-06-16T19:00:00Z". Le sort final compare les dates comme des chaînes de caractères — ça fonctionne car le format YYYY-MM-DD est naturellement triable alphabétiquement.

Fonction formatDayLabel(dateStr)

Convertit une date "2026-06-15" en texte lisible. Cas spéciaux pour aujourd'hui et hier, sinon format long.

"2026-06-16" → "Aujourd'hui" "2026-06-15" → "Hier" "2026-06-14" → "dimanche 14 juin"

864e5 c'est 864 × 10⁵ = 86 400 000 millisecondes = exactement 24h. On soustrait ça à Date.now() pour obtenir la date d'hier.

Composant ResultPanel({ results, loading })

Reçoit tous les résultats récents et gère la navigation entre les jours.

Bloc 1 — L'état de navigation

const grouped = groupByDay(results) const [dayIndex, setDayIndex] = useState(0)

grouped = les résultats organisés par jour. dayIndex = le jour actuellement affiché, 0 = le plus récent. Quand on clique sur la flèche gauche → dayIndex + 1 (on recule dans le temps).

Bloc 2 — Les données du jour actuel

const currentDay = grouped[dayIndex] const dayLabel = currentDay ? formatDayLabel(currentDay[0]) : null const currentMatches = currentDay ? currentDay[1] : [] const canGoBack = dayIndex < grouped.length - 1 const canGoForward = dayIndex > 0

currentDay[0] = la date du jour sélectionné (ex: "2026-06-15"). currentDay[1] = le tableau des matchs de ce jour. canGoBack = il reste des jours plus anciens à afficher. canGoForward = on n'est pas déjà sur le jour le plus récent.

Bloc 3 — Le rendu

Une barre de navigation avec deux boutons flèches SVG et le label du jour au centre. En dessous : skeleton si loading, message vide si aucun résultat, sinon les MatchCards des matchs du jour sélectionné.

📄 NewsCarousel.jsx

Affiche les actualités football en bas de l'accueil sous forme de carousel 3 par 3. Défile automatiquement toutes les 20 secondes, navigable manuellement avec des flèches et des points.

Constante PER_PAGE = 3

Nombre d'articles affichés par page du carousel. Changer cette valeur suffit à afficher 2 ou 4 articles par page.

Composant NewsCarousel({ news, loading, error })

Reçoit le tableau d'articles, un état de chargement, et une éventuelle erreur.

Bloc 1 — La pagination

const [page, setPage] = useState(0) const pages = Math.ceil(news.length / PER_PAGE) const slice = news.slice(page * PER_PAGE, page * PER_PAGE + PER_PAGE)

page = numéro de la page actuelle (commence à 0). pages = nombre total de pages (ex: 9 articles ÷ 3 = 3 pages). slice = les 3 articles à afficher pour cette page. Ex: page=1 → slice(3, 6) → articles 3, 4, 5.

Bloc 2 — Le défilement automatique

useEffect(() => { if (loading || pages <= 1) return const id = setInterval(() => setPage(p => (p + 1) % pages), 20000) return () => clearInterval(id) }, [page, pages, loading])

Toutes les 20 secondes, on avance d'une page. (p + 1) % pages fait tourner en boucle : après la dernière page on revient à la première. On ne lance pas le timer si les articles chargent encore ou s'il n'y a qu'une seule page. Le return () => clearInterval(id) arrête le timer quand le composant se démonte (nettoyage).

Bloc 3 — L'en-tête

Titre "Dernières actualités" + les points de navigation (un point par page). Le point de la page active a la classe --active pour être mis en évidence. Cliquer sur un point saute directement à cette page.

Bloc 4 — Le skeleton de chargement

Si loading est true → 3 cartes grises animées (image + source + titre + description). Même principe que PanelSkeleton dans MatchCard.jsx.

Bloc 5 — Le carousel d'articles

Deux boutons flèches ‹ › encadrent la grille d'articles. La flèche gauche est désactivée sur la première page, la droite sur la dernière. Chaque article est un lien <a> cliquable qui ouvre l'article dans un nouvel onglet. À l'intérieur : image (avec fallback si elle plante), source, titre, description, date et "Lire l'article →".

key={page} sur la grille force React à recréer le DOM à chaque changement de page, ce qui permet l'animation CSS de transition entre les pages.
